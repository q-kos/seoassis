1. ТЕХНИЧЕСКИЕ ПАРАМЕТРЫ
────────────────────────────────────────────────────────────────────────────────

Протокол и безопасность
  [ ] Сайт работает на HTTPS, HTTP автоматически перенаправляет на HTTPS
  [ ] Нет смешанного контента (mixed content) — все ресурсы загружаются по HTTPS
  [ ] SSL-сертификат действителен и не истекает в ближайшее время
  [ ] Настроены заголовки безопасности (X-Frame-Options, Content-Security-Policy и др.)

Скорость загрузки
  [ ] Время загрузки страницы не превышает 2–3 секунды
  [ ] Показатели Core Web Vitals в норме: LCP (загрузка), CLS (сдвиг макета), INP (отклик)
  [ ] Настроено кеширование браузера (Cache-Control, Expires)
  [ ] Включено сжатие Gzip или Brotli
  [ ] CSS, JavaScript и HTML минифицированы
  [ ] Изображения оптимизированы по размеру и весу, используются современные форматы (WebP, AVIF)
  [ ] Настроена отложенная загрузка (lazy loading) для изображений и видео
  [ ] Используется CDN для ускорения доставки контента (при необходимости)
  [ ] Время ответа сервера (TTFB) не превышает 200–300 мс

Мобильная версия
  [ ] Сайт корректно отображается на мобильных устройствах (адаптивный дизайн)
  [ ] Указан тег viewport в коде страниц
  [ ] Отсутствует горизонтальная прокрутка на мобильных устройствах
  [ ] Элементы интерфейса удобны для нажатия пальцем (размер кнопок и ссылок достаточный)

Код и сервер
  [ ] Чистый HTML-код без синтаксических ошибок
  [ ] CSS-стили вынесены в отдельные файлы, не используются inline-стили в избыточном количестве
  [ ] Нет JavaScript-ошибок в консоли браузера
  [ ] Страницы корректно отображаются при отключённом JavaScript (или используется SSR/пререндеринг)
  [ ] Нет ошибок сервера (500, 502, 503 и др.)
  [ ] Отсутствуют потенциальные уязвимости и вредоносный код в исходном коде


────────────────────────────────────────────────────────────────────────────────
2. URL И СТРУКТУРА САЙТА
────────────────────────────────────────────────────────────────────────────────

Формат URL
  [ ] URL-адреса страниц читаемы, понятны и содержат ключевые слова
  [ ] URL построены по единому принципу на всём сайте (латиница, один тип разделителя: / или _)
  [ ] В URL нет пробелов и спецсимволов (не используется кодировка %20 и подобные)
  [ ] Используются только строчные буквы — нет дублирования страниц на разных регистрах

Структура и вложенность
  [ ] Структура сайта логична: уровни вложенности соответствуют тематике разделов
  [ ] Важные страницы доступны не глубже 3–4 кликов от главной страницы
  [ ] Навигация сайта понятна пользователю
  [ ] Присутствуют хлебные крошки (breadcrumbs), отражающие корректную вложенность страниц


────────────────────────────────────────────────────────────────────────────────
3. ИНДЕКСИРОВАНИЕ И СКАНИРОВАНИЕ
────────────────────────────────────────────────────────────────────────────────

Robots.txt
  [ ] Файл robots.txt существует и доступен по адресу /robots.txt
  [ ] Закрыты от индексации служебные разделы: результаты поиска по сайту, корзина,
      личный кабинет, фильтры и сортировки, административная панель, страницы авторизации
      и регистрации, данные пользователей, кеш страниц, логи, версии для печати и т.п.
  [ ] В robots.txt не закрыты важные страницы, которые должны индексироваться
  [ ] В robots.txt указан путь к sitemap.xml

Sitemap.xml
  [ ] Файл sitemap.xml существует и корректно сформирован
  [ ] В sitemap включены только индексируемые страницы — без мусорных URL, дублей
      и страниц, запрещённых в robots.txt
  [ ] Sitemap передан в панели вебмастеров (Яндекс, Google)
  [ ] Sitemap обновляется при появлении новых страниц

Панели вебмастеров
  [ ] Сайт подтверждён и настроен в Яндекс Вебмастере
  [ ] Сайт подтверждён и настроен в Google Search Console
  [ ] В панелях вебмастеров нет ошибок сканирования и критических предупреждений
  [ ] Нет штрафных санкций или ручных фильтров со стороны поисковых систем
  [ ] Количество проиндексированных страниц соответствует ожидаемому

Ошибки и недоступные страницы
  [ ] Нет страниц, возвращающих ошибку 404 (кроме намеренно удалённых)
  [ ] Настроена корректная страница 404: есть понятное сообщение об ошибке и варианты
      действий (вернуться назад, перейти на главную, перейти в разделы)
  [ ] Нет ошибок сканирования в панелях вебмастеров


────────────────────────────────────────────────────────────────────────────────
4. РЕДИРЕКТЫ И ДУБЛИКАТЫ
────────────────────────────────────────────────────────────────────────────────

Редиректы
  [ ] Настроены редиректы 301: с www на без www (или наоборот — один канонический вариант)
  [ ] Настроен редирект с HTTP на HTTPS
  [ ] Настроен редирект со слешем на конце URL и без него — единообразно по всему сайту
  [ ] Главная страница перенаправляет все варианты (index.html, index.php и т.п.) на основной домен
  [ ] Нет цепочек редиректов (A → B → C) — только прямые (A → C)

Канонические страницы
  [ ] На страницах с возможными дублями указан тег canonical
  [ ] Тег canonical ссылается на корректный и индексируемый URL

Дубликаты
  [ ] Нет страниц-дублей с одинаковым или очень похожим контентом
  [ ] Нет дублей сайта на других доменах или в другой доменной зоне
  [ ] Нет дублирования метатегов (Title, Description, H1) на разных страницах


────────────────────────────────────────────────────────────────────────────────
5. МЕТАТЕГИ И РАЗМЕТКА
────────────────────────────────────────────────────────────────────────────────

Базовые метатеги
  [ ] На всех страницах заполнены Title и Description
  [ ] Title уникален для каждой страницы, содержит ключевые слова, длина 50–70 символов
  [ ] Description уникален для каждой страницы, длина 120–160 символов
  [ ] Тег meta robots не запрещает индексирование там, где это не нужно

Заголовки
  [ ] На каждой странице присутствует один тег H1, он уникален и не дублируется
  [ ] Присутствуют заголовки H2 и H3, они расположены последовательно без пропуска уровней
  [ ] Заголовки содержат ключевые слова и отражают содержание раздела

Микроразметка Schema.org
  [ ] Подключена микроразметка Schema.org: для карточек товаров, статей, организации,
      адреса, рейтингов, FAQ, событий и т.д. (в зависимости от типа сайта)
  [ ] Микроразметка корректна и проходит валидацию (Google Rich Results Test, Schema.org Validator)
  [ ] Используется формат JSON-LD (предпочтительный формат)
  [ ] Настроена разметка BreadcrumbList для хлебных крошек

Социальные и дополнительные теги
  [ ] Настроены Open Graph теги: og:title, og:description, og:image, og:url, og:type
  [ ] Настроены Twitter Cards
  [ ] Для многоязычных сайтов указан атрибут hreflang на всех страницах
  [ ] Присутствует фавикон (favicon), корректно прописан в коде страницы


────────────────────────────────────────────────────────────────────────────────
6. КОНТЕНТ
────────────────────────────────────────────────────────────────────────────────

Качество и уникальность
  [ ] Контент уникален, особенно на посадочных и коммерческих страницах
  [ ] Тексты написаны грамотно, без орфографических и пунктуационных ошибок
  [ ] На сайте нет страниц с минимальным бесполезным контентом (thin content)
  [ ] Содержание страниц актуально, регулярно обновляется
  [ ] Ключевые слова используются органично, без переспама (плотность в норме)

Изображения
  [ ] У всех изображений заполнен атрибут alt с релевантным описанием
  [ ] Изображения оптимизированы: сжаты, соответствующего размера, современные форматы
  [ ] Имена файлов изображений информативны (не img001.jpg, а например catalog-sofa-beige.jpg)

Внутренняя перелинковка
  [ ] Настроена грамотная внутренняя перелинковка между страницами
  [ ] Анкоры внутренних ссылок информативны и разнообразны
  [ ] Нет битых внутренних ссылок
  [ ] Нет «висящих» страниц без единой входящей внутренней ссылки

Внешние ссылки
  [ ] Внешние ссылки актуальны, нет битых внешних ссылок
  [ ] Ссылки на ненадёжные или нетематические ресурсы закрыты атрибутом nofollow

7. ССЫЛОЧНАЯ МАССА
────────────────────────────────────────────────────────────────────────────────

  [ ] Проведён анализ ссылочного профиля: качество, тематика и анкоры входящих ссылок
  [ ] Нет большого количества токсичных или спамных входящих ссылок
  [ ] При наличии вредоносных ссылок сформирован и загружен disavow-файл в панели вебмастеров
  [ ] Ссылочный прирост выглядит естественно, без резких искусственных скачков
  [ ] Разнообразие анкоров: брендовые, безанкорные, тематические ключевые слова

8. КОММЕРЧЕСКИЕ ФАКТОРЫ И ЛОКАЛЬНОЕ SEO
────────────────────────────────────────────────────────────────────────────────

Коммерческие факторы (важно для Яндекса)
  [ ] На сайте указаны контактные данные: телефон, адрес, email
  [ ] Присутствует страница «О компании» с реальной информацией
  [ ] Есть страница «Контакты» с адресом, картой и реквизитами
  [ ] Указаны цены, условия доставки и оплаты (для интернет-магазинов)
  [ ] Присутствуют пользовательские отзывы или возможность их оставить
  [ ] Сайт проверен на основные коммерческие факторы ранжирования Яндекса

Локальное SEO
  [ ] Зарегистрирован и полностью заполнен профиль в Яндекс Бизнесе
  [ ] Зарегистрирован профиль в 2GIS (при наличии офлайн-точки)
  [ ] Зарегистрирован и заполнен профиль в Google Business Profile
  [ ] Название, адрес и телефон (NAP) одинаковы во всех источниках и справочниках
  [ ] Сайт размещён в тематических каталогах и справочниках
